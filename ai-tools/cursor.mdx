Use Cursor to write and maintain documentation. This guide configures Cursor for technical writing with Mintlify components.

## Prerequisites

- Cursor editor installed
- Documentation repository access

## Project Rules

Create `.cursor/rules.md` in your repository root:

````markdown
# Mintlify Technical Writing Rule

AI assistant specialized in technical documentation using Mintlify components.

## Core Writing Principles

### Language and Style
- Clear, direct language for technical audiences
- Second person ("you") for instructions
- Active voice over passive
- Present tense for current states
- Define jargon when used
- Consistent terminology
- Concise sentences with context
- Parallel structure in lists/headings

### Content Organization
- Lead with most important information
- Basic concepts before advanced
- Break complex procedures into steps
- Include prerequisites and context
- Provide expected outcomes
- Descriptive, keyword-rich headings
- Logical grouping with clear sections

### User-Centered Approach
- Focus on user goals and outcomes
- Anticipate common questions
- Include troubleshooting for common failures
- Write for scannability
- Include verification steps

## Required Page Structure

Every page must have frontmatter:

```yaml
---
title: "Clear, specific title"
description: "Concise description of page purpose"
---
```

## Content Quality

### Code Examples
- Complete, runnable examples
- Proper error handling
- Realistic data (no placeholders)
- Expected outputs for verification
- Test before publishing
- Specify language and filename
- Add explanatory comments
- Never include real secrets

### API Documentation
- Document all parameters (including optional)
- Success and error response examples
- Rate limiting information
- Authentication examples
- HTTP status codes
- Complete request/response cycles

### Accessibility
- Descriptive alt text for images
- Actionable link text (not "click here")
- Proper heading hierarchy (start with H2)
- Keyboard navigation notes
- Sufficient color contrast
- Scannable structure

## Component Selection

- **Steps** for sequential procedures
- **Tabs** for platform-specific content
- **CodeGroup** for same concept in multiple languages
- **Accordions** for progressive disclosure
- **RequestExample/ResponseExample** for API endpoints
- **ParamField** for parameters, **ResponseField** for responses
- **Expandable** for nested objects
````

## Usage

After creating `.cursor/rules.md`, Cursor will:
1. Auto-apply writing standards to all docs
2. Suggest Mintlify components appropriately
3. Maintain consistent style
4. Follow technical writing best practices

## Tips

- Keep rules updated as documentation evolves
- Share rules file with team via git
- Review generated content for accuracy
- Customize rules for your project needs
