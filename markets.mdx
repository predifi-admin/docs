### Market Schedule

**All crypto markets operate 24/7:**

```
12:00 AM - 15M market opens
12:15 AM - Previous 15M resolves, new 15M opens
12:30 AM - Previous 30M resolves, new 30M opens
1:00 AM  - Previous 1H resolves, new 1H opens

... continues every day, all day
```

**That's:**
- **96** new 15M markets per day per asset
- **48** new 30M markets per day per asset
- **24** new 1H markets per day per asset

**With 6 assets:**
- **1,008 markets per day** across all time frames
- **~42 markets per hour**
- New opportunity every ~90 seconds

### How Markets Work

<Steps>
  <Step title="Market Opens">
    New market created with current price as reference
    
    ```
    Time: 2:00:00 PM
    BTC Price: $98,450 (reference price)
    Market: "BTC above $98,450 at 2:15 PM?"
    Initial Odds: YES 50% / NO 50%
    ```
  </Step>
  
  <Step title="Trading Period">
    Users trade based on predictions (15 min in this example)
    
    ```
    2:00-2:15 PM: Active trading
    - Price movements affect odds
    - Order book depth changes
    - Users can buy/sell anytime
    ```
  </Step>
  
  <Step title="Market Closes">
    No new trades allowed 30 seconds before resolution
    
    ```
    2:14:30 PM: Trading stops
    2:15:00 PM: Resolution time
    ```
  </Step>
  
  <Step title="Resolution">
    Oracle price feed determines outcome
    
    ```
    Resolution Price: $98,520
    Reference Price: $98,450
    Result: YES wins (price went up)
    ```
  </Step>
  
  <Step title="Settlement">
    Winners receive payouts automatically
    
    ```
    YES holders: Receive $1.00 per share
    NO holders: Shares become worthless
    ```
  </Step>
</Steps>

## Market Rules

### Resolution Criteria

**Crypto Markets:**

1. **Price Source:** Stork Network oracle (Binance, Coinbase, Kraken median)
2. **Resolution Time:** Exact time stated in market (e.g., 2:15:00 PM)
3. **Reference Price:** Price when market opened
4. **Resolution Price:** Price at resolution time
5. **Outcome:** YES if resolution > reference, NO if resolution ≤ reference

### Dispute Process

If you believe a market resolved incorrectly:

<Steps>
  <Step title="Review Resolution">
    Check oracle price data and resolution logic
  </Step>
  
  <Step title="Submit Dispute">
    File dispute within 24 hours of resolution
  </Step>
  
  <Step title="Investigation">
    Team reviews oracle data and market logic
  </Step>
  
  <Step title="Verdict">
    Decision made within 48 hours
  </Step>
  
  <Step title="Re-resolution">
    If dispute valid, market re-resolved and affected users compensated
  </Step>
</Steps>

- ✅ **Manipulation resistant** (40+ independent data sources)
