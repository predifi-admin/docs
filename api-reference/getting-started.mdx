## Welcome

The Predifi API provides programmatic access to prediction market trading. Build trading bots, custom UIs, analytics dashboards, or integrate markets into your application.

---

## Base URL

```
Production: https://api.predifi.com
Testnet:    https://testnet-api.predifi.com
```

All requests must use HTTPS.

---

## Authentication

Get your API key from the [Developer Dashboard](https://app.predifi.com/developer):

```bash
curl https://api.predifi.com/v1/markets \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### API Key Permissions

| Permission | Description |
|------------|-------------|
| **Read Markets** | Market data, orderbooks, prices |
| **Read Portfolio** | Positions and balances |
| **Place Trades** | Submit trading intents |
| **Cancel Orders** | Cancel pending orders |

<Warning>
  Never commit API keys to public repositories.
</Warning>

## Rate Limits

| Endpoint Type | Rate Limit | Burst |
|--------------|------------|-------|
| **Market Data (GET)** | 100 req/min | 200 req/min |
| **Trading (POST)** | 20 req/min | 40 req/min |
| **Portfolio (GET)** | 30 req/min | 60 req/min |
| **WebSocket** | 1 connection | - |

### Rate Limit Headers

```http
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1638360000
```

When rate limited:
```json
{
  "error": "RATE_LIMIT_EXCEEDED",
  "message": "Too many requests. Try again in 30 seconds.",
  "retryAfter": 30
}
```

---

## Response Format

### Success

```json
{
  "ok": true,
  "data": {
    // Response payload
  },
  "timestamp": 1638360000000
}
```

### Error

```json
{
  "ok": false,
  "error": "ERROR_CODE",
  "message": "Human-readable error",
  "details": {},
  "timestamp": 1638360000000
}
```

### Common Error Codes

| Code | HTTP | Description |
|------|------|-------------|
| `INVALID_API_KEY` | 401 | API key invalid |
| `INSUFFICIENT_PERMISSIONS` | 403 | API key lacks permission |
| `RATE_LIMIT_EXCEEDED` | 429 | Too many requests |
| `INVALID_PARAMS` | 400 | Invalid parameters |
| `MARKET_NOT_FOUND` | 404 | Market doesn't exist |
| `INSUFFICIENT_BALANCE` | 400 | Not enough USDC |
| `MARKET_CLOSED` | 400 | Market closed |

## Quick Start

<Steps>
  <Step title="Get API Key">
    [app.predifi.com](https://app.predifi.com) → Settings → Developer → Create API Key
  </Step>
  
  <Step title="Make First Request">
    <CodeGroup>
    ```bash cURL
    curl https://api.predifi.com/v1/markets \
      -H "Authorization: Bearer YOUR_API_KEY"
    ```
    
    ```typescript TypeScript
    import axios from 'axios';
    
    const api = axios.create({
      baseURL: 'https://api.predifi.com',
      headers: { 'Authorization': `Bearer ${process.env.PREDIFI_API_KEY}` }
    });
    
    const { data } = await api.get('/v1/markets');
    console.log(`Found ${data.data.length} markets`);
    ```
    
    ```python Python
    import requests
    
    headers = {'Authorization': f'Bearer {os.getenv("PREDIFI_API_KEY")}'}
    response = requests.get('https://api.predifi.com/v1/markets', headers=headers)
    print(f'Found {len(response.json()["data"])} markets')
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Place First Trade">
    ```typescript
    // Get quote
    const quote = await api.get('/v1/trading/quote', {
      params: {
        marketId: 'BTC-15M-20250111-1430',
        action: 'buy',
        outcomeId: 0,
        size: 100
      }
    });
    
    // Submit intent
    const intent = await api.post('/v1/trading/intent', {
      marketId: 'BTC-15M-20250111-1430',
      outcomeId: 0,
      size: 100,
      maxSlippage: 0.02,
      userAddress: '0x your_address'
    });
    
    console.log(`Intent: ${intent.data.intentId}`);
    ```
  </Step>
</Steps>

## API Endpoints

### Markets

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/v1/markets` | List all markets |
| `GET` | `/v1/markets/:id` | Get market details |
| `GET` | `/v1/markets/:id/prices` | Price history |
| `GET` | `/v1/markets/:id/trades` | Recent trades |

[Full documentation →](/api-reference/markets)

### Orderbook

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/v1/orderbook/:venue/:id` | Orderbook depth |
| `GET` | `/v1/orderbook/:venue/:id/top` | Best bid/ask |

[Full documentation →](/api-reference/orderbook)

### Trading

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/v1/trading/quote` | Get quote |
| `POST` | `/v1/trading/intent` | Submit intent |
| `GET` | `/v1/trading/intent/:id` | Check status |
| `DELETE` | `/v1/trading/intent/:id` | Cancel intent |

[Full documentation →](/api-reference/trading)

### Portfolio

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/v1/portfolio/:address/positions` | User positions |
| `GET` | `/v1/portfolio/:address/pnl` | P&L summary |
| `GET` | `/v1/portfolio/:address/history` | Trade history |

[Full documentation →](/api-reference/portfolio)

## Pagination

```bash
GET /v1/markets?page=1&limit=50
```

Response:
```json
{
  "ok": true,
  "data": [...],
  "pagination": {
    "page": 1,
    "limit": 50,
    "total": 1234,
    "pages": 25,
    "hasMore": true
  }
}
```

---

## Filtering & Sorting

```bash
# Filter by category
GET /v1/markets?category=crypto

# Sort by volume
GET /v1/markets?sort=volume&order=desc

# Combine
GET /v1/markets?category=crypto&status=active&sort=volume&order=desc
```

## Best Practices

<AccordionGroup>
  <Accordion title="Use WebSockets for Real-Time" icon="bolt">
    Don't poll REST endpoints. Use WebSocket for live updates.
  </Accordion>

  <Accordion title="Handle Errors Gracefully" icon="triangle-exclamation">
    Implement retries with exponential backoff.
  </Accordion>

  <Accordion title="Cache Aggressively" icon="database">
    Cache market metadata that changes rarely.
  </Accordion>

  <Accordion title="Use Pagination" icon="list-ol">
    Don't fetch all data at once. Paginate large lists.
  </Accordion>
</AccordionGroup>
