## Circle - USDC & CCTP

**Who:** Circle is the issuer of USDC ($35B+ market cap) and creator of CCTP (Cross-Chain Transfer Protocol).

**What We Use:**
- **USDC** - Primary stablecoin for all trading
- **CCTP** - Native USDC bridging between chains

**Why Circle:**
- **Native USDC** - Real USDC on all chains, not wrapped
- **Security** - Regulated, audited, fully backed 1:1 with USD
- **Speed** - 15-minute bridge times
- **Cost** - $0 bridge fees (only pay gas)

**How It Works:**
1. Burn USDC on source chain
2. Circle receives burn attestation
3. Mint native USDC on destination chain
4. No liquidity pools or wrapped tokens

**Supported Routes:**
- Arbitrum â†” Polygon, Base, Optimism
- Base â†” Polygon, Arbitrum, Optimism
- Polygon â†” Arbitrum, Base, Optimism

## Stork Network - Oracle Infrastructure

**Who:** Stork provides ultra-low-latency, manipulation-resistant oracle price feeds for DeFi.

**What We Use:**
- **Crypto price feeds** - BTC, ETH, SOL, and 50+ assets
- **Pull-based oracles** - On-demand data delivery
- **Real-time updates** - Sub-second price updates

**Why Stork:**
- **Low latency** - &lt;400ms from exchange to on-chain
- **Manipulation resistant** - 40+ independent publishers
- **Cost efficient** - Pay per use, not per update
- **Accurate** - Â±0.05% of true market price

**How It Works:**
1. Publishers fetch data from exchanges (&lt;100ms)
2. Stork aggregates from multiple sources (&lt;200ms)
3. Data delivered on-chain when requested (&lt;100ms)
4. Markets resolve using verified prices

**Key Features:**
- **Median aggregation** - Removes outliers automatically
- **Publisher incentives** - Staked reputation system
- **Fallback sources** - Multiple data providers
- **Historical data** - Full price history available

## Wormhole - Generic Bridging

**Who:** Wormhole is a generic cross-chain messaging protocol connecting 30+ blockchains.

**What We Use:**
- **Token bridging** - YES/NO position tokens
- **Message passing** - Cross-chain communication
- **Backup bridging** - When CCTP unavailable

**Why Wormhole:**
- **Battle-tested** - $10B+ in volume, 1B+ messages
- **Fast** - 1-2 minute finality
- **Secure** - 19 guardian network validators
- **Flexible** - Supports any token or message

**How It Works:**
1. Lock tokens on source chain
2. Guardian network validates
3. Mint wrapped tokens on destination
4. Redeem for native tokens later

**Use Cases:**
- Bridge position tokens back to user chains
- Emergency fallback for USDC bridging
- Future support for non-USDC assets

[Learn more about bridging â†’](/technology/cross-chain-bridging)

## Alchemy - Account Abstraction

**Who:** Alchemy provides Account Abstraction infrastructure powering the next generation of web3 wallets.

**What We Use:**
- **Smart Account Wallets** - For trade execution
- **Gas abstraction** - Gasless transactions
- **Bundler infrastructure** - Transaction batching

**Why Alchemy:**
- **Proven** - Powers 70% of top NFT projects
- **Scalable** - 400M+ requests per day
- **Reliable** - 99.99% uptime SLA
- **Developer friendly** - Best-in-class APIs

**How It Works:**
1. User signs trade intent
2. Protocol creates UserOperation
3. Alchemy Bundler executes on-chain
4. Gas fees handled by protocol

**Benefits for Users:**
- No gas tokens needed on execution chains
- Batch multiple trades in one transaction
- Social recovery for wallet access
- Session keys for temporary permissions
